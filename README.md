# kokudosekkei

このプロジェクトにおいて、AIエージェントは以下の「エンジニアの常識」を遵守すること。

1. 言語とコミュニケーション

すべてのチャット、コードコメント、ドキュメントは日本語で行うこと。

エラーメッセージは、システム例外をそのまま出すのではなく、以下の2点を含む日本語に加工すること。

何が起こったのか（例：ファイル形式が正しくありません）

ユーザーはどうすべきか（例：CSV形式のファイルを選択し直してください）

2. テスト駆動と品質保証

すべてのロジックに対して、ユニットテストを必ず作成すること。

テストは「ロジックのみを抽出した別ファイル」ではなく、リリース対象のコードそのものを対象に実行すること。

全条件パス（C1カバレッジ）を意識し、異常系・境界値のテストを含めること。

実装完了後、必ずテストを実行し、カバレッジの結果を報告すること。

3. 完了前のセルフ監査 出力前に以下のチェックリストを確認し、満たしていない場合は修正すること。

[ ] コード内にマジックナンバーや不明な英語の例外処理はないか？

[ ] 日本人ユーザーが読んで理解できるメッセージになっているか？

[ ] テストコードは、本番環境と同一の挙動を保証しているか？
